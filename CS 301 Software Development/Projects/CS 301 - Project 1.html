Motivation

The first project asks you to implement a solution to a chess problem, the so-called Knight's tour problem.

The knight's tour is a mathematical problem involving a single knight on a chess board. The knight is placed on the empty board and, moving according to the rules of chess, must visit each square exactly once. So on an 8 x 8 board, we have 64 squares and a valid tour is a sequence of 64 positions on the chess board which can be performed by a knight. 

A corresponding Wiki page mainly discusses the more special problem of a Closed Knight's Tour. Please use the wiki page only to better understand the problem of an OPEN knight's tour. The page has a different focus and may mislead you for this project assignment if you go too much through its details.


The open tour is a simpler problem. It does not require the knight to end at a position such that it could jump to the initial position next.

In other words, the open tour does NOT describe a cycle. 


Please follow a simple brute-force search. You can implement your solution a recursive or an iterative manner. Note that you can add as many additional methods to the class skeletons as you like. 

All you need to preserve and support is the externally visible (accessible) interface.

You can use these test cases for a test-driven development: Assume that you wrote the test cases yourself first. The initial (and basically empty) implementation of KnightsTourP1.java fails all the tests. In following steps you implement individual pieces of functionality in a piecemeal manner such that your code passes one more test in each step. Basically you use each test that the code fails as an intermediate goal to refine your implementation until it passes this test / this requirement. Once all tests are passed your are all set and your code fulfills all the requirements.
