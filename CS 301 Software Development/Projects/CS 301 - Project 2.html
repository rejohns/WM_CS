Motivation

There are many different ways to generate a Maze, for a nice summary click here and the section on Perfect Maze Creation Algorithms.


Falstad's code comes with one method that includes rooms. We would like to have more alternatives to generate a maze.

For this project, you are tasked to extend it by one further method, namely a randomized version of Prim's algorithm ( see section "Perfect Maze Creation Algorithm" here).

If you are not familiar with Prim's algorithm consult any computer science algorithms text book, wikipedia and in a variation for maze generation wikipedia.


What to learn from this project:

- Deriving test cases for an existing class (here: MazeBuilder) that is multi-threaded to understand its behavior and to be able to recognize if modifications corrupt corruptness

- Using inheritance to share code across classes that solve same problems in different ways

- Exercising a test-driven approach when extending an existing code base 

- Implementing an existing graph algorithm in Java



Design decision


The most straightforward design is to implement your new algorithm in a subclass of the existing MazeBuilder class. Since inheritance and abstract classes allow us to formulate generic solutions in a superclass and then have subclasses for more specific cases, let's exercise the following, slightly more challenging design.


To retain a reasonably uniform design, use the existing MazeBuilder class as a super class and create 2 subclasses named MazeBuilderFalstad and MazeBuilderPrim that inherit from it. 

The method that generates a maze according to a particular algorithm should be an "abstract" method in the super class and being provided by the subclasses. 


Of course, modifying an existing code base comes with the risk to introduce many subtle errors. 

To limit the risk and localize changes, please use a class design that starts with the MazeBuilder class and uses inheritance to share code among Falstad's maze generator and your implementation of Prim's algorithm.

Create a junit test class that exercises all functionality of the superclass to check that the computed maze has all properties of a correct maze, the provided solution encoded in the dists array is correct.
